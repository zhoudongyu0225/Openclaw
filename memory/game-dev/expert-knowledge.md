# 弹幕游戏 - 专家级知识库

## 产品定位

### 核心玩法
- **类型**：恐龙题材攻守塔防 + 双主播 PK
- **平台**：抖音、快手（国内）→ TikTok（海外）
- **特色**：实时双主播对抗，观众参与互动

### 战斗机制

#### 防守方视角（蓝色）
- 我方防御塔
- 我方基地（被恐龙破坏即失败）
- 敌方正生成的恐龙兵种（红色）
- 目标：阻止恐龙到达基地

#### 攻击方视角（红色）
- 我方恐龙生成点
- 我方恐龙部队
- 敌方防御塔和基地（蓝色）
- 目标：恐龙突破防线，摧毁基地

#### 胜负判定
- 任意一方基地被敌方恐龙破坏 = 失败
- 每局时长 3-5 分钟
- 多局制（BO3/BO5）

## 技术架构

### 整体架构
```
客户端（UE4）←→ 网关（Go）←→ 业务服务（Go）
                              ↓
                           存储（MongoDB + Redis）
                              ↓
                           第三方（抖音/快手 SDK）
```

### 核心模块

#### 1. 主播 PK 系统
- 双主播房间匹配
- 实时状态同步
- 观战系统
- 礼物打赏系统

#### 2. 战斗系统
- 塔防逻辑（防御塔属性、升级、攻击范围）
- 兵种系统（恐龙类型、属性、生产逻辑）
- 伤害计算
- 胜负判定

#### 3. 玩家系统
- 观众进入直播间
- 押注/竞猜
- 观战弹幕
- 礼物特效

#### 4. GM 后台
- 房间管理
- 主播管理
- 数据统计
- 运营配置
- 异常检测

#### 5. 模拟礼物工具
- 测试礼物特效
- 压力测试
- 自动化脚本

### 技术选型

| 模块 | 技术栈 | 原因 |
|------|--------|------|
| 游戏引擎 | UE4 | 渲染效果好，支持直播互动 |
| 后端 | Go | 高并发，性能好 |
| 协议 | Protobuf | 序列化效率高，体积小 |
| 存储 | MongoDB | 文档模型灵活，适合游戏数据 |
| 缓存 | Redis | 高速缓存，会话状态 |
| 实时通信 | WebSocket | 低延迟双向通信 |

## 盈利模式

### 收入来源
1. **礼物打赏**：观众送礼物，主播分成
2. **门票/参赛费**：主播 PK 参赛费用
3. **广告**：游戏内原生广告
4. **增值服务**：特效、皮肤、VIP

### 分成比例
- 平台：50%
- 主播：40%
- 游戏方：10%

## 运营策略

### 冷启动阶段
1. **找种子主播**：游戏主播、泛娱乐主播
2. **内容营销**：短视频切片、直播切片
3. **福利拉新**：初期给主播高分成

### 增长阶段
1. **赛事运营**：举办主播 PK 大赛
2. **跨平台推广**：抖音→快手→TikTok
3. **UGC 激励**：用户创作内容

### 成熟阶段
1. **国际化**：多语言适配
2. **电竞赛事**：职业化
3. **IP 衍生**：周边、联名

## API 设计

### 核心接口

#### 房间管理
```protobuf
// 创建房间
rpc CreateRoom(CreateRoomReq) returns (CreateRoomRes)

// 加入房间
rpc JoinRoom(JoinRoomReq) returns (JoinRoomRes)

// 离开房间
rpc LeaveRoom(LeaveRoomReq) returns (LeaveRoomRes)
```

#### 战斗系统
```protobuf
// 开始战斗
rpc StartBattle(StartBattleReq) returns (StartBattleRes)

// 同步状态
rpc SyncBattleState(SyncReq) returns (SyncRes)

// 操作防御塔
rpc OperateTower(OperateTowerReq) returns (OperateTowerRes)

// 生成恐龙
rpc SpawnDinosaur(SpawnReq) returns (SpawnRes)
```

#### 礼物系统
```protobuf
// 发送礼物
rpc SendGift(SendGiftReq) returns (SendGiftRes)

// 礼物特效
rpc GiftEffect(GiftEffectReq) returns (GiftEffectRes)
```

## 风险与对策

| 风险 | 对策 |
|------|------|
| 平台政策变化 | 多平台布局，不依赖单一平台 |
| 主播流失 | 主播成长体系 + 利益绑定 |
| 技术延迟 | 边缘计算 + 预加载 |
| 抄袭复制 | 专利保护 + 快速迭代 |
| 合规风险 | 法务审核 + 敏感词过滤 |
