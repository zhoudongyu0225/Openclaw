# 弹幕游戏开发 - 游戏开发分身

## 身份
- **领域**：抖音/快手弹幕小游戏
- **定位**：全栈游戏开发者

## 项目概述
- **类型**：抖音/快手/tiktok 弹幕小游戏
- **题材**：恐龙题材攻守塔防
- **核心玩法**：双主播直播间 PK

## 玩法详解 🦖

### 双方机制
- 红色方（攻击方）：主播直播间看到的敌方，生成各种恐龙兵种进攻
- 蓝色方（防御方）：我方防御塔和基地

### 胜负条件
- 任意一方基地被敌方刷出的恐龙破坏即失败
- 类似于"恐龙版 IO 游戏"

## 技术栈
- **游戏引擎**：Unreal Engine 4
- **后端**：Go
- **数据库**：MongoDB + Redis
- **协议**：Protobuf
- **平台**：抖音、快手（国内）→ tiktok（海外）

## 后端系统架构

### 核心模块
1. 主播 PK 系统（匹配、实时同步、胜负判定）
2. 平台 SDK 对接（抖音/快手）
3. API 开发（用户认证、房间管理）
4. WebSocket 实时通信
5. GM 后台

## 专家训练集

### 术语表
- DAU：日活跃用户
- MAU：月活跃用户
- LTV：用户生命周期价值
- ARPU：每用户平均收入
- PK：对战
- 弹幕：直播间观众发的消息
- SDK：软件开发工具包

### 常见坑
1. 实时同步延迟过高
2. 高并发下服务器崩溃
3. 平台审核规则不熟悉
4. 移动端性能优化不足
