# AI 理解压缩器

## 目的
将自然语言理解和中间推理转换成结构化的 md 格式，提高后续处理效率。

## 输入格式
人类自然语言

## 输出格式 (md)

```markdown
# 意图理解

## 用户意图
[一句话概括用户想要什么]

## 任务类型
- 分类: [speed/smart/text/unknown]
- 复杂度: [低/中/高]
- 是否需要新会话: [是/否]

## 关键实体
| 实体 | 值 | 来源 |
|------|-----|------|
| 用户 | @白老师 | 系统 |
| 任务 | xxx | 用户输入 |

## 上下文关联度
- 与当前会话: [高/中/低]
- 原因: [简短说明]

## 待执行动作
1. [动作1]
2. [动作2]

## 风险评估
- 风险点: [无/有]
- 级别: [低/中/高]
```

## 使用方法

### 在执行任务前，先调用理解器生成结构化 md

```python
# 伪代码
understanding = await compress_understanding(
    user_input="帮我看看今天有啥安排",
    session_context=current_context
)
# understanding 已经是 md 格式
# 后续可以：存档/路由/执行/反馈
```

### 场景应用

1. **任务路由**：识别后自动路由到对应模型池
2. **知识归档**：结构化存入记忆系统
3. **会话管理**：判断是否开启新会话
4. **审计追溯**：完整记录理解过程
